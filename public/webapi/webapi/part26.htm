10.1 OAuth2.0

El API Expedientes-dealerBest emplea OAuth2.0 Client Credentials para identificar (autenticar) y autorizar a los usuarios del API. Aplicaciones han de estar autenticadas y autorizadas antes de intercambiar datos con Expedientes-dealerBerst


10.1.1 Roles y definiciones

Los nombre empleados en la RFC y su significado en este API:

• Resource owner – El usuario propietario de la cuenta en Expedientes dealerBest que quiere acceder a sus datos mediante el API

• Resource server – El servidor donde reside la aplicación de Expedientes dealerBest y se ejecuta el API. https://expedientes.dealerbest.com

• Client – El sistema informático, aplicación, código o persona que realice los http Request contra el API.

• Authorization server – En nuestro caso se trata del mismo servidor donde reside la aplicación de Expedientes dealerBest.

• Client ID – Un código identificador único que identifica la aplicación (Client) que quiere utilizar el API.

• Cliente secret – Un código único que solo conoce el creador del Client ID.

• Authorization code – Código empleado durante el proceso OAuth para demostrar que la autorzación ha sido otorgada.

• Access token – Un token que es requerido para accedera cualquier función del API.

• Refresh token – un token que puede ser utilizado una vez para obtener un nuevo Access token y un nuevo Refresh token.
